# WeaveForge — 织造坊

> 跨宇宙 TRPG 跑团知识管理系统设计。  
> 一套完整的数据库设计方案，支撑跨世界角色穿越、正典一致性校验、跑团日志自动转化为小说故事。

---

## 核心理念

```
需求（project-design.md）→ 实现（arcadedb-schema.md）→ 代码（DDL + access 模块）
      ↑ 数据库无关                    ↑ 绑定 ArcadeDB 26.5+
```

**两层分离**：
- **项目设计文档**：定义系统需要什么数据域、支持什么查询。数据库无关。
- **Schema 设计文档**：将项目设计映射到 ArcadeDB 的具体实现。

## 设计覆盖

| 领域 | 内容 |
|------|------|
| **跨宇宙通道** | 维度桥接(实时)、平行世界(实时)、跨作品航行(需载具+航行时间) 三种类型 |
| **角色系统** | 28字段四层档案(身份/能力/跨世界/叙事)，纯标签属性系统，正典/原创/同人三分 |
| **认知模型** | 世界层(社会阶层认知模板)+角色层(个人背景)+经历层(跑团见闻)，含可信度和认知冲突检测 |
| **偏离级联** | 事件偏离后自动计算人物/政治/剧情三级影响传播，支持侧面级偏离 |
| **路径规划** | ROUTE边(时间成本/交通方式/危险等级) + Vehicle顶点(速度/航程/跃迁)，图遍历路径组装 |
| **跨世界桥接** | 物品/能力穿越时的三层判定(通道→兼容性→认知映射) |
| **小说生成** | 每会话自动触发，三种预设风格模板(第一人称日记/第三人称史诗/轻小说) |
| **Oracle规则** | Mythic GME 风格，5级概率表，纯Oracle驱动无骰子 |

---

## 数据库依赖

- **ArcadeDB 26.5+**（`LSM_VECTOR` + `vector.fuse` + `GEOSPATIAL`）
- **LLM**：本地 `qwen3:8b` / `qwen3:14b`（Ollama）+ 云端 DeepSeek / GPT
- **向量模型**：`bge-m3`（Ollama），1024 维 COSINE

## 变更流程

```
需求变化
  → 第一步：更新 project-design.md（确认数据域/查询能力）
  → 第二步：更新 arcadedb-schema.md（映射到数据库实现）
  → 第三步：更新 init 脚本（DDL 对齐）
  → 第四步：必要时更新 access 模块（查询接口对齐）
```
